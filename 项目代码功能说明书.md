## 一、项目架构总览

```
Qt-Gaming
├── main.cpp          → 程序入口，创建 QApplication 和 Game 窗口
├── game.h/cpp        → 游戏主控类：窗口、键盘输入、游戏循环、地图切换、传送门逻辑
├── player.h/cpp      → 玩家类：贴图、移动、墙壁碰撞检测
├── tilemap.h/cpp     → 地图管理类：加载瓦片、管理墙壁列表、提供碰撞检测接口
├── tile.h/cpp        → 瓦片类：最基本的图片显示单元（地板/墙/门等）
├── maploader.h/cpp   → 地图加载器：解析 Tiled 导出的 JSON (.tmj) 地图文件
└── resources.qrc     → Qt 资源文件：注册所有图片和地图文件
```

**数据流**：
```
Tiled 地图(.tmj) → MapLoader 解析 → TiledMapData 结构体 → TileMap 创建 Tile 对象 → 加入 QGraphicsScene 渲染
                                           ↓
                                    提取出生点、传送门数据
                                           ↓
                                    Game 类创建 Player、处理键盘输入、碰撞检测、传送逻辑
```

| 73-80 | `collidesWithWall()` | **碰撞检测实现**：<br>遍历 `walls` 列表中的每个墙壁瓦片，调用 `item->collidesWithItem(wall)` 检测是否相交。<br>一旦碰撞立即返回 `true`，否则遍历完返回 `false`。 |

#### 6.2 maploader.cpp

| 行号 | 代码 | 功能说明 |
|------|------|----------|
| 10 | `static const int GID_MASK = 0x1FFFFFFF` | Tiled 使用 GID 的高 3 位存储水平翻转、垂直翻转、对角翻转标志，低 29 位是实际图块 ID。此掩码用于提取纯 ID。 |
| 12-25 | `MapLoader::load()` 开头 | 打开 JSON 文件，读取全部内容，解析为 `QJsonDocument`。失败返回 `false`。 |
| 28-31 | 读取地图基本尺寸 | `width`、`height`、`tilewidth`、`tileheight` |
| 34-49 | **解析 tilesets（图块集）** | 遍历 `root["tilesets"]` 数组：<br>- 读取 `firstgid`（图块集的起始 GID）<br>- 读取 `image`（图块集图片路径）<br>- 将路径转换为 Qt 资源路径格式：`:/images/文件名`<br>- 存入 `firstGidToImage[firstGid] = imagePath` |
| 52-62 | Lambda `getImageForGid` | **根据 GID 查找所属图块集**：<br>遍历所有 `firstGid`，找到小于等于目标 GID 且最大的那个（即目标 GID 所属的图块集），返回其图片路径。<br>⚠️ **假设**：每个图块集只有一张图片（即所有瓦片都在一张 spritesheet 里）。 |
| 65-91 | **解析 tilelayer（瓦片图层）** | 遍历 `root["layers"]`：<br>- 类型为 `"tilelayer"` 的才处理<br>- 读取 `data` 数组，将每个原始 GID 与 `GID_MASK` 做与运算清除标志位<br>- 存入 `outMap.layerData[layerName]`<br>- 同时为每个非 0 GID 记录图片路径到 `gidToImage` |
| 92-127 | **解析 objectgroup（对象层）** | 类型为 `"objectgroup"` 的图层：<br>- 遍历所有对象<br>- **如果对象名 == `"start"`** → 设为玩家出生点<br>- **如果图层名 == `"portals"` 且矩形宽高 > 0** → 创建 Portal：<br>  - `id = objName`<br>  - `rect = QRectF(x, y, w, h)`<br>  - 读取对象的 `properties` 数组，提取 `target_map` 和 `target_portal_id`<br>  - 加入 `outMap.portals` |
| 131-134 | 默认图片兜底 | 如果某个 GID 没有映射到图片，默认设为 `:/images/floor.png` |
| 136 | `return true` | 加载成功 |
