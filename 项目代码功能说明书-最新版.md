## 一、项目架构总览

```
Qt-Gaming-main-v3
├── main.cpp              → 程序入口，创建 QApplication 和 Game 窗口
├── game.h/cpp            → 游戏主控类：窗口、键盘输入、游戏循环、地图切换、
│                           传送门、技能系统、敌人管理、HUD、宠物、巢穴、交互系统
├── player.h/cpp          → 玩家类：GIF动画、移动、碰撞、HP/MP/EXP、等级进化、
│                           变身形态、施法动画
├── tilemap.h/cpp         → 地图管理类：加载瓦片、墙壁/水碰撞（空间索引网格）
├── tile.h/cpp            → 瓦片类：图片显示单元，支持固定尺寸缩放
├── maploader.h/cpp       → 地图加载器：解析 Tiled JSON (.tmj) 地图文件
├── skill.h/cpp           → 技能投射物类：火球/简易弹/破空梭/月牙弹/刀浪/GIF刀浪
├── enemy.h/cpp           → 敌人类：随机游走 AI、三方向炮弹攻击、血条显示
├── pet.h/cpp             → 宠物类：跟随玩家、Idle/Run 状态切换、DFS 绕墙
├── spawner.h/cpp         → 巢穴类：定时生成怪物波次、血量倍率递增
└── resources.qrc         → Qt 资源文件：注册所有图片和地图文件
```

### 数据流

```
Tiled 地图(.tmj) → MapLoader 解析 → TiledMapData 结构体
                                            ↓
                              Game::loadMap() 手动绘制所有图层
                                            ↓
                              创建 Tile、Enemy、Spawner、Pet 对象
                                            ↓
                              Player 接收键盘输入 → move() → 碰撞检测
                                            ↓
                              updateGame() 60FPS 主循环：
                                - 玩家移动 + 动画
                                - 投射物更新 + 碰撞伤害
                                - 敌人 AI + 攻击
                                - 巢穴计时生成
                                - 宠物跟随
                                - HUD 更新
                                - 传送门检测
                                - 宝箱/门交互
```

##### 构造函数（第 90-109 行）

```cpp
Game::Game() {
    scene = new QGraphicsScene(this);
    setFixedSize(800, 600);
    // 预加载帧缓存（避免首次释放技能卡顿）
    preloadProjectileFrames();

    loadFireBgFrames();
    loadDaolangFrames();
    loadMap(":/maps/school_map.tmj", true);

```

##### 析构函数（第 111-197 行）

清理所有动态分配的资源：投射物、盾牌、宠物、敌人、敌人炮弹、巢穴、HUD 元素、tileMap、player。采用安全遍历删除模式。

##### `loadMap()` 函数（第 199-590 行）—— **最复杂的函数**

这是整个项目最长、最重要的函数，完整流程如下：

```
1. 暂停游戏循环（stop timer）
2. 清理所有现有资源（tileMap, player, projectiles, enemies, spawners, pet, HUD, etc.）
3. 清空场景中所有项
4. 创建新 TileMap，调用 MapLoader 解析 JSON
5. 手动绘制所有图层（二次解析 JSON）：
   ├── minion/minion_image → 创建 Enemy 对象（含血条）
   ├── water → 创建水瓦片（加入水碰撞索引）
   ├── fireland → 记录火焰区域矩形 + 创建显示瓦片
   ├── floor → 随机 floor_dark/floor_light 纹理
   ├── wall → 随机 wall_1/wall_2/wall_3 纹理 + 加入墙壁碰撞
   ├── chest → 加入宝箱列表
   ├── door → 加入门列表 + 加入墙壁碰撞
   ├── boss/boss_image/elite → 加入可攻击列表（hittableItems）
   └── 其他装饰层 → 按名称映射图片
6. 创建玩家（连接死亡信号）
7. 创建背后火焰（如已解锁）
8. 创建宠物
9. 创建 3 个巢穴（基于出生点偏移）
10. 连接升级信号
11. 设置玩家位置（出生点或留空）
12. 动态计算场景矩形（根据地图实际尺寸）
13. 重置缩放 + 创建 HUD
14. 重新启动游戏循环
```

**关键改进**（相比 v1.0）：
- 场景矩形从硬编码 `12800×6400` 改为动态计算 `mapWidth*tileWidth × mapHeight*tileHeight`
- Y 坐标 bug（`y * tileWidth` → `y * tileHeight`）已修复
- 地图切换时不再有场景残留（`loadGameTimer stop` 后完整清理再重建）

##### `updateGame()` 主循环（第 632-734 行）

每帧（16ms）执行顺序：
```
1. 变身动画期间暂停 → return
2. 闪现动画处理（每帧移动一步，生成残影，结束射出刀浪）
3. 正常移动（含水碰撞回退）
4. HP/MP 恢复（每 20 帧 +1）
5. 更新所有投射物（火球/简易弹/月牙弹/破空梭/刀浪/GIF刀浪）
6. 更新玄武盾位置
7. 地形效果检测（火焰扣血）
8. 更新宠物位置
9. 更新背后火焰动画
10. 更新所有敌人
11. 更新所有敌人炮弹
12. 更新所有巢穴
13. 更新 HUD
14. 传送门检测
15. 宝箱/门交互
```

##### 技能系统详解

**I 键 — 天火燎原（`skillMeteorBurst()`，第 755-798 行）**
- 条件：静止时（四方向键均未按下）+ 消耗 10 MP
- 1-2 级：发射 8 方向红色椭圆子弹（SimpleProjectile，伤害 25）
- 3 级+：发射 8 方向 GIF 火球（Projectile，伤害 25）
- 变身：附带 `player_enhanced_fly_fire.gif` 施法动画

**J 键 — 普攻1 九重炎杀（`skillNormalAttack()`，第 1148-1208 行）**
- 条件：无消耗，随时可用
- 以玩家为中心 96×96 范围攻击（伤害 15）
- 特效：`fire_hit.gif` 火焰打击 GIF 持续 2 秒
- 变身：附带 `player_enhanced_pugong.gif` 施法动画

**K 键 — 瞬影浪斩（`skillFlashBlade()`，第 1018-1085 行）**
- 条件：有方向输入 + 消耗 15 MP
- 步进法计算闪现路径（遇墙停止，最大 100px）
- 5 帧跨帧动画，每帧生成红色残影
- 闪现终点射出刀浪（伤害 40）：
  - 1 级：蓝色矩形刀浪
  - 2 级+：GIF 刀浪（`daolang_left.gif`，支持 8 方向旋转）
- 变身：附带 `player_enhanced_K.gif` 普攻动画

**N 键 — 普攻2 月牙八刃（`skillBlueBurst()`，第 800-824 行）**
- 条件：无消耗，可移动发射
- 8 方向月牙形蓝色子弹（BlueProjectile，伤害 15×8）

**H 键 — 普攻3 熔岩弹（`skillTriangleShot()`，第 950-964 行）**
- 条件：无消耗
- 朝移动方向发射 1 枚三角形子弹（伤害 45）
- 默认朝向：无方向输入时朝右

**L 键 — 玄武盾（`skillShieldActivate/Deactivate()`，第 1210-1243 行）**
- 消耗 5 MP 开启
- 圆形护盾（半径 28px），跟随玩家
- 可阻挡敌人炮弹

**爆炸效果（`createExplosion()`，第 966-993 行）**
- 5 级解锁
- 预加载 `bomb.gif` 帧（每 2 帧取 1 帧，缩放到 96×96）
- QTimer 8ms 手动切帧，约 0.38 秒播完

##### 敌人系统（第 1393-1520 行）

**`updateEnemies()`**（第 1393-1415 行）
- 根据玩家等级动态调整敌人攻击间隔：`max(30, 120 - (level-1)*15)`
- 遍历所有敌人，调用 `enemy->update()`
- 死亡敌人：从 hittableItems 移除 → 给玩家 +20 EXP → delete

**`updateEnemyProjectiles()`**（第 1496-1520 行）
- 遍历所有敌人炮弹
- 检测是否击中玩家 → 玄武盾阻挡或 `player->takeDamage(10)`

##### 传送系统（第 1522-1623 行）

**`performTeleport()` 改进**：
- 使用 `hitboxRect()` 检测（而非 `sceneBoundingRect()`）
- `safeTeleportTo` 防卡墙：先计算基准位置，尝试 8 方向 32px 偏移
- 同地图传送冷却 2 秒，跨地图冷却 5 秒
- 传送后宠物自动重置

##### 交互系统（第 1676-1806 行）

**宝箱**（`openChest()`）：
- 玩家碰撞检测 → +0.25 钥匙 → 金色闪光特效（200ms）→ 移除宝箱

**门**（`removeDoorRegion()`）：
- 扩展检测范围（+tileSize 各方向）
- BFS 遍历四个方向连通的门瓦片
- 从碰撞系统移除 → 场景删除 → 消耗 1 钥匙
- 开门特效（绿色光圈 200ms）

##### 死亡重生（第 1625-1647 行）

- 玩家 HP 归零 → 发射 `died()` 信号
- `onPlayerDied()`：`player->reset()` → 传送到出生点 → 重置冷却 → 摄像头对准

#### 3.2 关键方法

**`move(up, down, left, right)`**
- 计算 dx/dy → 更新朝向 → 切换动画状态 → 轴分离碰撞（先 X 后 Y）

**`hitboxRect()`**
- 1 级：返回完整 32×32 碰撞框
- 2 级+：返回右下角 32×32 碰撞框（因为显示为 64×64）

**`addExp(amount)`**
- 等级上限 20 级
- 升级时：HP/MP 上限 ×1.5，瞬间回满，发射 `levelUp` 信号
- 经验阈值每级 ×1.2

**`reset()`**
- 等级→1，HP/MP→100，经验→0，关闭增强形态，刷新动画

**`setEnhanced(bool)`**
- 切换增强形态，强制刷新 GIF 切换（idle/run 均替换为 enhanced 版本）

**`playCastAnimation(gifPath, frameInterval)`**
- 变身形态下按技能键播放施法帧动画
- 预加载所有帧 → 逐帧播放 → 播完恢复正常动画

**`updateCastAnimation()`**
- 每帧切换施法帧，支持朝向翻转

**`updateAnimationState(moving, right)`**
- 根据移动/静止、增强/普通状态切换对应 GIF：
  - 静止普通：`player.gif`
  - 移动普通：`player_run.gif`
  - 静止增强：`player_enhanced.gif`
  - 移动增强：`player_enhanced_right_run.gif`

#### 4.1 墙壁碰撞——空间索引网格优化

```
gridPixelSize = tileSize * GRID_SIZE (32 * 8 = 256px)
每个格子独立存储墙壁列表
碰撞检测时只检查玩家所在格子及相邻格子
```

| 方法 | 说明 |

### 5. tile.h / tile.cpp

相比 v1.0 的变化：

- 构造函数新增 `fixedSize` 参数（`QSize`，可选）

### 6. maploader.h / maploader.cpp

与 v1.0 基本相同，核心功能不变：

#### 数据结构

| 结构体 | 说明 |
|--------|------|
| `Portal` | 传送门：id、触发矩形、目标地图路径、目标传送门 id |

```

                              └── objectgroup → 提取 start 出生点 + portals 传送门
```

**注意**：`GID_MASK = 0x1FFFFFFF` 用于清除 Tiled 的翻转/旋转标志位。
