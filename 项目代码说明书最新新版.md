## 一、项目架构总览

```
Qt-Gaming-main
├── main.cpp              → 程序入口，创建 QApplication 和 Game 窗口
├── game.h/cpp            → 游戏主控类 (~2900行)：窗口、键盘输入、游戏循环、地图切换、
│                           传送门、全部技能、敌人管理、HUD、宠物、巢穴、菜单系统、
│                           钻石系统、小地图、梅花瓣粒子、地图介绍、管理员模式
├── player.h/cpp          → 玩家类 (~298行)：GIF多状态动画、移动、碰撞、HP/MP/EXP、等级进化、
│                           变身形态、施法动画、跨地图状态保持
├── tilemap.h/cpp         → 地图管理类 (~174行)：加载瓦片、墙壁/水碰撞（空间索引网格）
├── tile.h/cpp            → 瓦片类 (~25行)：图片显示单元，保留原始分辨率 + transform 缩放
├── maploader.h/cpp       → 地图加载器 (~137行)：解析 Tiled JSON (.tmj) 地图文件
├── skill.h/cpp           → 技能投射物类 (~487行)：火球/简易弹/破空梭/月牙弹/刀浪/GIF刀浪
├── enemy.h/cpp           → 敌人类 (~310行)：随机游走 AI、8种GIF外观随机、三方向炮弹攻击、血条
├── pet.h/cpp             → 宠物类 (~138行)：跟随玩家、Idle/Run 状态切换、11方向DFS绕墙
├── spawner.h/cpp         → 巢穴类 (~85行)：定时生成怪物波次、血量倍率递增、安全放置算法
├── resources.qrc         → Qt 资源文件：注册 ~150 个图片/地图/GIF文件
└── Qt-Gaming.pro         → Qt 项目配置文件：widgets模块、console输出
```

### 构建系统

- **构建工具**：qmake（Qt-Gaming.pro）
- **Qt 版本**：Qt 5/6（widgets 模块）
- **编译**：`qmake && make -j$(sysctl -n hw.ncpu)`
- **产物**：`Qt-Gaming.app`

### 数据流

```
Tiled 地图(.tmj) → MapLoader 解析 JSON → TiledMapData 结构体
                                            ↓
                              Game::loadMap() 二次解析 JSON 手动绘制所有图层
                                            ↓
                              创建 Tile、Enemy、Spawner、Pet、钻石、传送门精灵
                                            ↓
                              Player 接收键盘输入 → move() → 碰撞检测(墙+水)
                                            ↓
                              updateGame() 60FPS 主循环：
                                - 闪现动画处理
                                - 定身状态检查
                                - 玩家移动 + 水碰撞回退
                                - HP/MP 恢复 (每20帧)
                                - 投射物更新 + 碰撞伤害
                                - 玄武盾位置更新
                                - 地形效果 (火焰扣血)
                                - 宠物更新
                                - 背后火焰动画
                                - 敌人 AI + 攻击 + 死亡奖励
                                - 敌人炮弹更新 + 玩家受伤
                                - 巢穴计时生成
                                - HUD 更新
                                - 传送门旋转 + 检测
                                - 宝箱/门交互
                                - 地图介绍检测
                                - 小地图更新
                                - 钻石动画/碰撞
                                - 梅花瓣粒子
                                - 攻击buff计时器
```

##### loadMap() 函数（第 511-1183 行）—— 最核心函数

这是整个项目最长、最重要的函数。完整流程：

```
1. 停止 gameTimer
2. 保存跨地图状态 (savedLevel/Exp/Hp/Mp/Enhanced)
3. 删除旧 player（先保存状态再删）
4. 清理所有资源：
   - 投射物(6种)、盾牌、敌人/炮弹、巢穴、宠物
   - 变身动画QMovie（关键：先stop再delete，防野指针崩溃）
   - 背后火焰、HUD(全部8个)、小地图
   - 钻石列表、传送门精灵列表、buffIndicator
   - 清空 hittableItems/chests/doors/fireRects
   - 重置 keyCount=0、attackBuffTimer=0
```

## 三、关键数据流与算法

### 3.1 碰撞检测流程

```
玩家移动(move)
  ├── setPos(x+dx, y)
  ├── collidesWithWall(hitboxRect())
  │   └── 计算网格坐标范围
  │       └── 遍历 gridWalls[{gx,gy}]
  │           └── wall->sceneBoundingRect().intersects(rect)
  │   → 回退 setPos(x-dx, y)
  ├── setPos(x, y+dy) → 同上
  └── collidesWithWater(hitboxRect()) → 回退到原位置
```

### 3.2 手动绘制图层顺序

在 `Game::loadMap()` 中按 JSON layers 顺序遍历：

```
minion/minion_image → Enemy创建
water → Tile(Z=-3) + 水碰撞
fireland → Tile(Z=-4) + fireRects
floor/floor_onfire/floor_road/floor_room → Tile(Z=-2)
wall → Tile(Z=-1) + 墙碰撞
stair_h/c1/c2 → Tile(Z=-1)
grass/grassland → Tile(Z=-4)
chest → Tile(Z=0) + chests列表
door → Tile(Z=-1) + doors列表 + 墙碰撞
portal/portal_image → 跳过(用漩涡替代)
boss/boss_image/elite → Tile(Z=0) + hittableItems
其他 → Tile(Z=0)
```

### 3.3 传送门安全传送 safeTeleportTo()

```
targetCenter - (48, 48) → basePos → player->setPos(basePos)
↓
collidesWithWall(hitboxRect())? → Yes:
  尝试8方向各偏移32px →
  找到不撞墙的位置 → 使用
  全部失败 → 退回 basePos
```

### 3.4 钥匙系统 BFS 开门

```
removeDoorRegion(startDoor):
├── BFS队列遍历：
│   └── 对每个 door: 检查上下左右4邻居
│       └── 相同 grid 坐标 + 在 doors 列表中 → 入队
├── 从 tileMap 碰撞系统移除所有 visited doors
├── 从 scene 删除所有 visited doors
└── 从 doors 列表移除
```
